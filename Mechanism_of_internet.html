HOW THE INTERNET WORKS: AN OVERVIEW

★ THE BASIC INFRASTRUCTURE:

At a high level, the Internet is a global network of interconnected computers and systems that communicate seamlessly through a standard ruleset called protocols.

🌐 The Backbone: Interconnected Routers

The absolute core of this structure is a massive, decentralized grid of routers. Their single job is directing information traffic.

When you transmit data, it is instantly chopped into tiny chunks called packets. Packets do NOT follow one fixed line; they flow dynamically through whatever paths are fastest!

💡 Figure 1.0: Packet Swapping & Routing Pathway
💡 EXAM TERMS REFERENCE:
  • 📌 Packet: Smallest unit of data transmitted over networks.
  • 📌 Router: Device directing packets between different networks.
  • 📌 IP Address: Unique identifier assigned to every online device.
  • 📌 DNS: System translating domain names into IP addresses.
💡 Examiner's Tip: IP routes packets, while TCP guarantees correct order delivery! Remember this contrast!

Topic: Protocols, IP & DNS (02/05) Page 2

ROLE OF PROTOCOLS, IP & DNS

★ PROTOCOLS: THE GLUE

A Protocol is an agreed set of rules and standards defining how information is exchanged. Without them, devices from different companies couldn't communicate!

📌 IP Addresses vs. Domain Names

Computers find each other using IP Addresses. An IP is a unique numerical label.

• IPv4 Address: E.g., 192.168.1.1 (Four numbers divided by dots).

• IPv6 Address: Hexadecimal format for modern vast scales.

Humans cannot easily memorize numeric strings. Therefore, we use domain names (e.g., google.com) to alias numeric IPs.

💡 Figure 2.0: Domain Name System Translation Loop

The DNS Server is the "Phonebook of the Internet". When you enter a domain, your browser sends a DNS query, fetches the numeric address, then negotiates content directly with the Web Server.

THE THREE PILLARS & HYPERLINKS

★ TIM BERNERS-LEE'S THREE PILLARS (1989)

The World Wide Web stands securely on three major pillars:

  1. 👉 1. URL: An address system keeping track of documents.
  2. 👉 2. HTTP: The transfer protocol to fetch these documents.
  3. 👉 3. HTML: The document structure supporting hyperlinks.
🔗 Hyperlinks: The Web's Catalyst

Before the Web, finding files was difficult. Hyperlinks revolutionized everything by correlating a text string with a precise URL, letting users jump instantly across files!

📂 INTERNAL LINKS:

Connects two webpages belonging to the same website. Key to smooth navigation.

📂 EXTERNAL LINKS:

Connects your webpage to someone else's website. Essential to linking the global web.

• Incoming Links: Links pointing *to* your site from external domains (critical for Search Engine Optimization!).

• Anchors: Tie sections of one single document together. Clicking jumps immediately inside the same page without refreshing.

ANATOMY OF A URL & SEMANTICS

★ WHAT IS A URL?

A URL (Uniform Resource Locator) is the unique address of a published document or media file on the web.

🔬 Anatomy Breakdown:
https://www.example.com:80/path/file.html?key=val#section
Scheme
HTTPS / HTTP
Domain
example.com
Port
80 / 443
Path
/path/file.html
Params & Anchor
?key=val & #section
📬 Real-World Analogy (Postal Mail):
  • 📬 Scheme: The delivery postal service used.
  • 📬 Domain Name: The city or town of destination.
  • 📬 Port: Zip code (technical gateway).
  • 📬 Path: The building where mail is dropped off.
  • 📬 Anchor: The specific recipient room inside.
🌱 Semantic URLs (Best Practice)

Always use words with inherent meaning (e.g., .../Learn_web_development) rather than a soup of random characters! They improve accessibility, usability, and classification of websites in search engines.

WEB SERVERS, INTRANETS & EXTRANETS

★ WEB SERVERS: HARDWARE vs. SOFTWARE

A Web Server refers to two things working together:

• 1. Hardware: A dedicated, always-on computer storing your files (HTML docs, CSS, media).

• 2. Software: At a minimum, an HTTP Server that processes incoming requests and delivers pages.

👉 Static Web Server: Delivers hosted files exactly "as-is" to your browser.

👉 Dynamic Web Server: Updates files "on-the-fly" using databases and template engines before sending them out.

  • 🚀 Extranets: Opens a portion of private networks to trusted partners (suppliers, customers).
